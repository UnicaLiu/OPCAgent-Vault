# AGENTS.md — OPCAgent 专用子代理

> 本文件定义可被 spawn 的专用 sub-agent。行为规则见 [[CLAUDE.md]]。

---

## product-ingestor

- **用途**：摄入新产品原始资料，创建产品档案
- **触发**：`raw/产品原始资料/` 出现新文件时
- **流程**：
  1. 读取 raw/产品原始资料/ 中的产品文件（含图片）
  2. 在 `wiki/产品库/产品档案/` 创建 `{产品名}.md`
  3. 提取：名称、卖点、价格、SKU、目标国家、分类、禁忌偏好
  4. 建立与公共专家库中相关策略/规则的 wikilinks
  5. 更新 `wiki/index.md` 产品索引
- **输出**：产品档案 .md + index.md 更新

---

## report-generator

- **用途**：为指定产品生成 AI 产品分析报告
- **触发**：产品档案创建/更新后，或手动触发
- **流程**：
  1. 读取产品档案 + raw/ 原始资料
  2. 读取 `wiki/公共专家库/` 中相关平台规则和投放策略
  3. 生成版本化报告：`wiki/产品库/产品报告/{产品名}-产品报告-v{N}.md`
  4. 报告含：市场定位、目标受众、卖点分析、内容策略建议、风险提示
  5. 更新产品档案的 `report_versions` 字段
  6. 新报告自动激活（is_active: true），旧版本标记 inactive
- **输出**：产品报告 .md + 产品档案更新

---

## hot-product-analyzer

- **用途**：分析爆款视频/产品，提炼可复用策略
- **触发**：`raw/爆品素材/` 有新素材，或定期执行
- **流程**：
  1. 读取 raw/爆品素材/ 中的视频描述、截图、数据
  2. 在 `wiki/爆品分析/分析报告/` 生成分析报告
  3. 提取：视频结构、文案公式、视觉风格、音乐/BGM、互动模式
  4. 更新 `wiki/爆品分析/素材索引/` 的特征标注
  5. 如发现新打法，更新 `wiki/公共专家库/内容方法论/`
- **输出**：分析报告 .md + 素材索引更新

---

## task-recorder

- **用途**：记录和管理 OPCAgent 执行的任务
- **触发**：OPCAgent 后端创建新任务时
- **流程**：
  1. 在 `wiki/任务中心/任务记录/` 创建 `task-{task_id}.md`
  2. 记录 config_snapshot（完整任务参数冻结）
  3. 任务执行中更新状态（PENDING → RUNNING → SUCCESS/FAILED）
  4. 关联产品档案、报告版本、生成的脚本
  5. 失败时记录 failure_reason + 堆栈 + 恢复建议
- **输出**：任务记录 .md + 状态更新

---

## script-archiver

- **用途**：归档生成的脚本，关联任务和产品
- **触发**：脚本生成完成时
- **流程**：
  1. 在 `wiki/任务中心/脚本库/` 创建脚本文件
  2. 关联分镜宫格图引用
  3. 记录 prompt_snapshot 和生成参数
  4. 建立与产品档案、任务记录的双向链接
- **输出**：脚本 .md

---

## attribution-analyzer

- **用途**：分析投放数据归因，提炼数据洞察
- **触发**：新的投放数据可用时，或定期执行
- **流程**：
  1. 读取 raw/ 中的投放数据
  2. 在 `wiki/数据归因/归因报告/` 生成归因分析
  3. 提炼关键洞察到 `wiki/数据归因/数据洞察/`
  4. 如有可反哺产品认知的结论，更新产品报告或产品档案
- **输出**：归因报告 .md + 数据洞察 .md

---

## wiki-linter

- **用途**：知识库定期健康检查
- **规则参照**：`CLAUDE.md` → 第五节 5.3 Lint
- **检查项**：
  - 断裂 wikilinks（指向不存在页面）
  - 孤儿页面（无入链、无出链）
  - index.md 与实际文件是否同步
  - frontmatter 完整性和格式
  - 内容矛盾与过时信息
  - 值得新建的汇总/对比页
- **输出**：修复建议 + 自动修复断裂链接

---

## knowledge-syncer

- **用途**：多主机部署时的知识同步
- **触发**：手动或定时
- **职责**：
  - 比较本机 `wiki/公共专家库/` 与远端版本
  - 同步公共专家库的更新
  - 上报本机的归因洞察（不含敏感产品数据）
- **输出**：同步报告

### Agent 开发规范

**核心模式**：每个 Agent = 独立 Web UI + Claude CLI 子进程。

| 原则 | 说明 |
|------|------|
| **Web UI 即 Agent** | 每个页面是独立 Agent，有自己的界面和职责 |
| **Claude CLI 即大脑** | 不配 API Key，用 `claude -p --bare` 子进程调用 |
| **调度层越薄越好** | 不预处理、不预搜索，直接把用户输入 + System Prompt 交给 Claude |
| **告诉它，不要替它** | System Prompt 下指令（"去 wiki/ 找相关页面"），不预加载数据 |
| **文件即状态** | Agent 输出直接写入 Vault，天然持久化 |
| **端口即身份** | 每个 Agent 独立端口（8765+），可独立启停 |
| **MCP 即知识网关** | 所有 wiki/ 读写通过 MCP，确保 index/log 自动维护 |

### 开发新 Agent 步骤

1. 在本注册表新增一行
2. 复制已有 Agent 页面代码作为模板
3. 替换 System Prompt（引用 `[[System-Prompt-模板库]]` 中对应的模板）
4. 改 UI → 分配新端口（8765+，避免冲突）
5. 配置 `.claude/mcp.json` 指向本 Vault 的 MCP Server
6. 启动验证 `wiki_stats` 能正常返回
