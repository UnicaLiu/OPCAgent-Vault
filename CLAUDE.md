# CLAUDE.md — OPCAgent 量化知识库

> 这是 Claude 在 OPCAgent Vault 中的行为配置文件。
> 每次会话开始：先读本文件 → 了解知识库结构 → 开始工作。

---

## 一、知识库定位

本 Vault 是 **OPCAgent 电商量化跑品系统** 的知识库，部署在多个本地主机上。

以「**产品**」为唯一业务源头，串联完整量化链路：

```
产品原始资料 → 公共专家知识 → 产品分析报告 → 爆品视频分析 → 任务执行 → 数据归因反馈
     ↑                                                              │
     └──────────────── 归因结果反哺产品认知 ───────────────────────────┘
```

**核心隐喻**：本知识库 = OPCAgent 的「数字认知伙伴」。它帮 Agent 记忆、整理、发现关联，让每次执行都在上一次的基础上更聪明。

## 二、架构：三层分离

```
OPCAgent Vault/
├── CLAUDE.md                    ← 本文件：行为规则 + 知识库结构
├── AGENTS.md                    ← 子代理定义（可 spawn 的专用 agent）
├── raw/                         ← 原始资料层（只读，人类投喂）
│   ├── 产品原始资料/             ← 产品图片、规格书、供应商文档
│   ├── 竞品参考/                 ← 竞品信息、市场分析
│   ├── 爆品素材/                 ← 爆款视频素材、创意参考
│   └── 外部知识/                 ← 网页剪藏、论文、外部文档
├── wiki/                        ← 知识库层（Agent 维护，结构化）
│   ├── index.md                 ← 中央导航（Agent 自动维护）
│   ├── 公共专家库/               ← 共享的专家知识（平台规则/投放策略/内容方法论/工具指南）
│   ├── 产品库/                   ← 产品档案 + AI 生成的版本化产品报告
│   ├── 爆品分析/                 ← 爆款视频分析报告 + 素材索引
│   ├── 任务中心/                 ← 任务记录 + 生成的脚本
│   ├── 数据归因/                 ← 投放数据归因 + 数据洞察
│   └── 对比分析/                 ← 跨产品/跨策略对比
├── 知识库基础搭建原理/           ← 知识库理论参考（只读）
└── .obsidian/                   ← Obsidian 配置
```

**三层职责**：

| 层 | 读写权限 | 维护者 | 用途 |
|---|---------|--------|------|
| `raw/` | 只读 | 人类 | 原始资料投放区，Agent 只读取不修改 |
| `wiki/` | 读写 | Agent | 结构化知识，Agent 负责创建、更新、维护 |
| 配置层 | 直接编辑 | 人类 | CLAUDE.md / AGENTS.md / .obsidian/ |

## 三、核心规则

### 3.1 写入规则

| 场景 | 规则 |
|------|------|
| **新建 wiki 页面** | 必须包含 YAML frontmatter（title / tags / created / type） |
| **更新 wiki 页面** | 保留已有 frontmatter，更新 modified 字段 |
| **产品报告** | 版本化命名：`{产品名}-产品报告-v{N}.md`，生成后更新产品档案的 `report_versions` |
| **任务记录** | 以 `task_id` 为主键，记录完整 config_snapshot |
| **raw/ 文件** | **禁止修改** raw/ 下的任何文件，只读取 |
| **index.md** | 每次新增/删除 wiki 页面后更新 index.md 的目录结构和统计 |

### 3.2 命名约定

| 场景 | 规则 | 示例 |
|------|------|------|
| 产品档案 | `{中文产品名}.md` | `智能桌面种植机.md` |
| 产品报告 | `{中文产品名}-产品报告-v{序号}.md` | `智能桌面种植机-产品报告-v1.md` |
| 任务记录 | `task-{task_id}.md` | `task-tk-20260614-001.md` |
| 脚本文件 | `{产品名}-脚本-v{序号}.md` | `智能桌面种植机-脚本-v1.md` |
| 分析报告 | `{主题}-分析报告-{日期}.md` | `美妆类目-分析报告-2026-06-14.md` |
| 归因报告 | `{产品名}-归因报告-{日期}.md` | `智能桌面种植机-归因报告-2026-06-14.md` |
| 概念/方法 | `{概念名}.md` | `AIDA公式.md` |
| 工具指南 | `{工具名}-使用指南.md` | `Claude-Code-使用指南.md` |

### 3.3 Frontmatter 规范

每个 wiki 页面必须包含以下 frontmatter：

```yaml
---
title: "页面标题"
type: "product | product_report | task | script | analysis | attribution | concept | tool | rule | comparison"
tags: ["标签1", "标签2"]
created: 2026-06-14
modified: 2026-06-14
status: "draft | active | archived"
sources: ["raw/xxx.md"]  # 关联的原始资料来源
---
```

### 3.4 链接规范

- **跨 wiki 引用**：使用 wikilinks `[[产品库/产品档案/产品名]]`
- **引用 raw/**：使用 wikilinks `[[raw/产品原始资料/文件]]`
- **外部链接**：使用 markdown 链接 `[文本](url)`
- **图片嵌入**：使用 `![[image.png]]`（图片放在对应 raw/ 目录）

## 四、五大板块详解

### 板块 1：公共专家库 `wiki/公共专家库/`

Agent 共享的知识底座，**在所有部署主机上保持一致**。

```
公共专家库/
├── 平台规则/        ← TikTok / Shopee / Amazon 等平台的广告政策、算法规则
├── 投放策略/        ← 跑品策略、出价方法、受众定向、素材策略
├── 内容方法论/      ← 脚本公式（AIDA/PAS/...）、视频结构、文案框架
├── 工具指南/        ← OPCAgent 工具使用、AI CLI 配置、环境搭建
└── 术语表.md        ← 电商/投放/量化领域术语
```

**更新策略**：公共专家库由核心团队维护，定期同步到各部署主机。

### 板块 2：产品库 `wiki/产品库/`

```
产品库/
├── 产品档案/        ← 每个产品一个文件，包含完整产品信息
└── 产品报告/        ← AI 生成的版本化产品分析报告
```

**产品档案包含**：基础信息、卖点、价格、SKU、目标国家、分类、禁忌偏好、关联报告索引。

**产品报告**：每次生成新版本自动激活（is_active: true），旧版本保留但标记 inactive。

### 板块 3：爆品分析 `wiki/爆品分析/`

```
爆品分析/
├── 分析报告/        ← TikTok/电商平台爆款视频分析、市场趋势
└── 素材索引/        ← 爆款视频特征标注、可复用的素材元素索引
```

### 板块 4：任务中心 `wiki/任务中心/`

```
任务中心/
├── 任务记录/        ← 每个任务的 config_snapshot、执行状态、产物索引
└── 脚本库/          ← 生成的脚本（含分镜宫格图引用）
```

### 板块 5：数据归因 `wiki/数据归因/`

```
数据归因/
├── 归因报告/        ← 投放数据归因分析（ROI / 转化路径 / 受众效果）
└── 数据洞察/        ← 从归因数据中提炼的认知和策略建议
```

## 五、工作流

### 5.1 摄入新资料（Ingest）

当人类投喂原始资料到 `raw/`：

1. **读取** raw/ 中的目标文件（含图片）→ 获取完整信息
2. **识别** 资料类型（产品/竞品/爆品素材/外部知识）
3. **编译** 到 wiki/ 对应板块：
   - 产品资料 → 创建/更新 `产品库/产品档案/` + 触发报告生成
   - 竞品参考 → 创建/更新竞品对比信息
   - 爆品素材 → 创建 `爆品分析/分析报告/` + 更新 `素材索引/`
   - 外部知识 → 判断归属板块，创建对应 wiki 页面
4. **关联** 建立 wikilinks，连接相关概念、产品、策略
5. **更新** `wiki/index.md` 索引

### 5.2 查询知识（Query）

1. 读取 `wiki/index.md` 了解知识库全貌
2. 按需深入具体板块的 README.md
3. 通过 wikilinks 追溯相关知识节点
4. 综合回答，标注来源页面

### 5.3 健康检查（Lint）

定期执行：
- 检查 wikilinks 是否有断裂（指向不存在页面）
- 检查是否有孤儿页面（无入链、无出链）
- 检查 index.md 与实际文件结构是否一致
- 检查 frontmatter 完整性
- 标记过时/矛盾内容

## 六、与 OPCAgent 系统的关系

本 Vault 对应 OPCAgent 项目的 `knowledge/` 目录，是系统的知识持久化层。

```
OPCAgent 后端 (FastAPI + Celery)
    │
    ├── 读取 wiki/ 获取产品认知 → 生成脚本
    ├── 写入 wiki/ 保存报告/任务/脚本/归因
    │
    ▼
OPCAgent Vault (本知识库)
    ├── raw/      ← 人类投喂原始资料
    ├── wiki/     ← Agent 生成和维护的知识
    └── CLAUDE.md ← Agent 行为配置
```

**关键原则**：
- OPCAgent 后端通过文件系统直接读写本 Vault
- AI CLI（Claude Code / Codex）子进程以本 Vault 为工作目录
- 每次 AI 调用自动加载 CLAUDE.md 获取上下文
- 知识积累形成飞轮：越用越聪明

## 七、部署说明

本 Vault 设计为**可在多台本地主机上独立部署**：

| 内容 | 部署方式 |
|------|---------|
| 目录结构 + 配置文件 | Git 版本管理，统一分发 |
| `wiki/公共专家库/` | Git 同步，核心团队维护 |
| `wiki/产品库/` | 主机独立，各自跑品不同产品 |
| `wiki/任务中心/` | 主机独立，本地任务记录 |
| `raw/` | 主机独立或共享素材库 |
| `.obsidian/` | Git 忽略（本地 Obsidian 偏好） |

**主机初始化步骤**：
1. `git clone` 本 Vault
2. 配置 Obsidian 打开此目录
3. 投放产品原始资料到 `raw/产品原始资料/`
4. 启动 OPCAgent，Agent 自动开始摄入和编译
