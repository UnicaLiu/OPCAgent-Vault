# raw/ — 原始资料层

> **规则**：raw/ 是只读的资料投放区。Agent 只读取，不修改。所有整理后的知识写入 `wiki/`。

---

## 目录结构

| 目录 | 用途 | 接受的文件类型 |
|------|------|--------------|
| `产品原始资料/` | 产品图片、规格书、供应商文档、卖点说明 | .md / .png / .jpg / .pdf / .json |
| `竞品参考/` | 竞品信息、市场分析报告、价格对比 | .md / .png / .pdf |
| `爆品素材/` | 爆款视频截图、创意参考、热门视频链接 | .md / .png / .jpg / .mp4 |
| `外部知识/` | 网页剪藏、论文、行业报告、工具文档 | .md / .pdf |

---

## 文件命名规范

| 场景 | 规则 | 示例 |
|------|------|------|
| 产品资料 | `{产品名}-{资料类型}.{ext}` | `智能桌面种植机-产品图.png` |
| 网页剪藏 | 保持 Clipper 自动命名（日期-标题） | `2026-06-14-TikTok广告新规.md` |
| 竞品资料 | `{竞品名}-{资料类型}.{ext}` | `绿植精灵-定价对比.json` |
| 爆品素材 | `{日期}-{平台}-{描述}.{ext}` | `2026-06-14-TikTok-爆款美妆视频.md` |

---

## 投放流程

```
人类投放 raw/ 文件
    │
    ▼
Agent 自动检测（或手动触发 ingest）
    │
    ├── 产品资料 → product-ingestor → wiki/产品库/
    ├── 爆品素材 → hot-product-analyzer → wiki/爆品分析/
    ├── 竞品参考 → 综合分析 → wiki/产品库/ + wiki/对比分析/
    └── 外部知识 → 判断归属 → wiki/相应板块
```

---

## 注意事项

- ⚠️ **Agent 不会删除 raw/ 中的文件**，处理完成后可手动清理
- ⚠️ 图片文件请使用 PNG/JPG 格式，确保 Agent 能通过 vision 读取
- ⚠️ 大文件（>50MB 视频）建议外链存储，raw/ 中只放描述 .md
- ✅ 可以使用 Obsidian Web Clipper 一键保存网页到 `外部知识/`
- ✅ 同名产品的新资料直接放入，Agent 会对比版本差异

---

*raw/ 层的设计受 Karpathy LLM-Wiki 模式启发。原材料 → 编译 → 结构化知识的飞轮从这里启动。*
